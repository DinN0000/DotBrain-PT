수집 다음 단계: AI가 활용 가능한 구조 만들기

앞선 세션이 자료를 생성하고 수집하는 단계였다면, 이 세션은 그 다음 단계입니다.
수집한 문서를 AI가 이해하고, 연결하고, 근거로 활용할 수 있게 만드는 단계입니다.

자료 생성/수집
↓
지식베이스 구조화
↓
AI 의사결정 품질 향상
문서 간 상호 관계가 표현된 지식베이스가 들어가면, AI는 컨텍스트 메모리를 덜 쓰면서도 필요한 지식을 정확히 참조합니다.

현장 사례

같은 프롬프트여도 결과가 달라진다

실제 사용 예시

신한은행 제안 준비로 만들었던 발표자료가 지식베이스에 정리되어 있었고,
"신한은행 발표자료를 참고해서 세미나 발표자료 만들어줘"
짧은 프롬프트만으로도 초안을 빠르게 만들 수 있었습니다.

지식베이스가 약할 때

검색 중심 결과. 단편적 인용. 맥락 누락.

지식베이스가 연결돼 있을 때

넓은 맥락 참조. 근거 일관성. 결과물 품질 향상.

핵심은 "더 많은 문서"가 아니라, AI가 읽을 수 있는 구조와 관계를 가진 문서입니다.

문제

효과는 알지만, 모두가 쓰기엔 어렵다

개인 단위에서는 정리 습관이 문제이고, 조직 단위에서는 운영 복잡도가 문제입니다.

개인

정리의 마찰

유형 분류, 폴더 구조 설계, 문서 연결 설정을 매번 수동으로 해야 함

비개발자

높은 진입장벽

정해진 규격과 메타데이터 규칙을 계속 맞추기 어려움

조직

운영 난이도 증가

Single Source of Truth와 개인 더미 데이터가 섞이며 품질 관리 비용 증가

결과

유지 실패

초기 구축은 해도 시간이 지나면 지식베이스가 다시 무너짐

DotBrain 정의

개인이 로컬에서 가장 쉽게 운영하는 지식베이스

문제 정의: 사내 지식베이스를 만들기 전 단계에서, 개인이 로컬 환경에서 지식베이스를 쉽게 운영하게 만들자.
Human-Readable (사람이 읽고 유지하기 쉬움)
+
AI-Optimized (AI가 구조와 관계를 즉시 파악)
=
운영 가능한 개인 지식베이스

피해야 할 극단 1

RAG 중심으로만 설계해 사람이 읽기 어려운 구조

피해야 할 극단 2

사람 친화 구조만 두어 AI가 검색 이상으로 못 가는 상태

구조 선택

Human Layer는 PARA를 사용한다

PARA는 "문서 형식"이 아니라 "시간에 따른 쓰임"으로 분류합니다. 개인이 계속 유지하기 쉬운 이유입니다.

Project

마감이 있는 현재 과제

Area

지속적으로 책임지는 영역

Resource

필요 시 참조하는 자료

Archive

완료/비활성 콘텐츠

DotBrain은 PARA를 버리지 않습니다. 사람이 익숙한 구조를 유지한 채, AI 레이어를 자동으로 쌓습니다.

핵심 설계

같은 문서, 사람과 AI가 다르게 읽게 만든다

사람에게는 노트 경험을 유지하고, AI에게는 메타데이터와 관계 그래프를 제공합니다.

레이어누가 사용무엇이 보이는가생성 주체
Human Layer 사용자 마크다운 본문, 폴더, 위키링크 사용자
AI Layer AI 에이전트 Frontmatter, MOC, note-index, related notes DotBrain 자동 생성
구조화를 사용자에게 강요하지 않고, AI가 필요로 하는 구조를 자동 생성하는 것이 DotBrain의 차별점입니다.

작동 방식

사용자 행동은 단순하게, 구조화는 자동으로

Drop — 파일을 _Inbox/에 넣는다
↓
Organize — 분류 + Frontmatter + 목차 갱신
↓
Connect — 관련 노트 추천 + 링크 삽입
↓
Maintain — 태그 정규화 + 누락 필드 보충 + 품질 점검

사용자가 하는 일

쓰기, 저장, 가끔 승인

DotBrain이 하는 일

분류, 구조화, 연결, 정리 자동화

핵심 기능

4가지 장치로 운영 부담을 줄인다

01

Auto Sort + Frontmatter

인박스 입력만으로 PARA 분류, tags/summary/status 자동 생성

02

Wiki-links + Semantic Linking

노트 간 관계를 자동 발굴해 Related Notes로 연결

03

MOC + AI Companion

폴더 컨텍스트를 문서화해 AI 온보딩 시간을 단축

04

Self-Healing

태그 정규화, 누락 필드 보충, 중복/품질 점검 자동 실행

핵심 목표는 기능 추가가 아니라, 지식베이스 유지 비용을 줄여 계속 쓰게 만드는 것입니다.

운영 관점

개인에서 팀으로 확장 가능한 운영 모델

Local First
개인이 로컬에서 바로 사용
→
Human-in-the-Loop
AI 추천, 최종 결정은 사용자
→
Quality Guardrails
자동 점검 + 점진적 학습
→
Org Ready
신뢰 가능한 공유 지식베이스로 확장
시작은 개인 생산성 개선이지만, 구조가 쌓이면 팀과 조직의 지식 자산으로 자연스럽게 이어집니다.

마무리

기대 효과

지식 활용률 향상

정리되지 않아 사장되던 문서가 PARA 구조 안에서 자동으로 연결되고 재사용됩니다. "쓰고 잊어도 찾아지는" 개인 지식베이스를 만듭니다.

AI 결과 품질 개선

AI가 폴더 맥락과 문서 관계를 함께 읽으므로, 같은 프롬프트에서도 더 깊고 일관된 결과를 만듭니다.

운영 비용 절감

Self-Healing과 승인 기반 자동화로 품질 유지 부담을 줄여, 개인과 조직 모두 지속 가능한 운영이 가능합니다.

앞으로

확장 방향: Local → Domain → Global

개인 로컬 환경에서 시작해, 도메인 단위, 전사 단위로 확장하는 3계층 구조를 제안합니다.

Local Layer
개인 지식베이스
→
Domain Layer
SCOPE/FLAP/AX 등 팀별 도메인 지식
→
Global Layer
전사 공통 지식과 공유 표준
DotBrain은 개인의 작업 환경을 그대로 두고, 그 위에 AI 최적화 레이어를 더했습니다. 사용자는 기존처럼 문서를 작성하면 되고, DotBrain이 분류·메타데이터 생성·문서 연결·품질 점검을 자동으로 수행합니다. 그 결과 문서는 사람이 읽기 쉬우면서(Human-readable), AI가 즉시 활용할 수 있는(Optimized for AI) 지식으로 전환됩니다.
Outlook: 조직의 투자 의지가 뒷받침된다면 Local ↔ Domain ↔ Global 순으로 지식이 확장·순환되며, 사용할수록 데이터가 축적되고, 데이터가 모일수록 모델이 정교해지고, 모델이 정교해질수록 활용이 확대되는 선순환을 만들 수 있습니다. 이 선순환이 다음 단계에서 조직이 만들어가야 할 과제라고 생각합니다.
