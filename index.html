DotBrain · AI 세미나

15분 발표자료

“지식은 쌓였는데, AI는 다시 쓰지 못한다”

오늘의 핵심: 기능 설명이 아닌 지식운영 구조로 설득한다.

발표 포인트 3개

  • 왜 지금까지는 인사이트가 재사용되지 않는가
  • 왜 기존 방식은 시간이 갈수록 무너지는가
  • 개인 KB → 조직 KB로 넘어가는 현실적인 길

발표 시간: 15분 / 12장 / 1가지 메시지 집중

첫 2분: 출발점

앞선 세션을 기억하시면 핵심은 이미 알고 계십니다

  • 리서치 자동화 세션에서 다량의 자료를 생성
  • 자료를 지식베이스에 넣으면 AI 재활용이 쉬워질 것이라 기대
  • 그런데 실제로는 “재요청했을 때 매번 새로 시작”되는 느낌이 큼

한 문장으로

문제는 AI가 나쁘다는 게 아니라, 조직화된 지식이 부실하다는 것.

문제 3분

실제 병목은 ‘난이도’다

에이전트 실행

요청은 쉽다. 출력도 나온다.

운영

수집·태깅·링크·중복·유효성 관리가 반복되어 비용 폭증

사내 확장

개인산출물 + 검증되지 않은 산출물의 신뢰도 구분이 어렵다

결과

같은 주제를 다시 물어도 매번 제로베이스

이 메시지가 공감되지 않으면, 아무 기능 소개도 의미가 약해집니다.

해결 프레임

DotBrain은 “도구”보다 운영 방식이다

철학

기존 파생 산출물을 버리지 않는다. 구조만 부드럽게 추가한다.

진입 전략

개발 지식이 없어도 시작. _Inbox/ 중심의 흐름

인증 포인트

“내가 직접 정해줄 규칙”이 아니라 “내가 쓰는 방식”에 적응

확장

개인 KB로 검증 후, 팀 공유 구조로 점진적 확대

핵심 주장 1 (4~6분)

Auto Sort: 진입 비용 제거

지금 방식:

• 폴더 판단 → 이동 → 태그 → 링크

DotBrain 방식:

• _Inbox/로 넣기 → AI가 PARA/메타데이터 자동 반영

발표에서 바로 말할 한 줄

“정렬을 하느라 논의하던 시간을, 분석과 판단으로 되돌린다.”

장점: 진입장벽 급감, 재시작 비용 급감

핵심 주장 2 (6~8분)

자동 구조화: AI가 즉시 이해하는 지식베이스

자동 생성

Frontmatter / MOC / Semantic Links

동작

요청 시 전체 검색이 아니라, 인덱스로 후보 선별 후 정밀 조회

효과

같은 주제 반복 질의에서 응답 일관성이 올라감

한 줄 정리

같은 자료도, 읽는 방식이 바뀌면 결과가 바뀐다.

핵심 주장 3 (8~10분)

Self-Healing: 시간이 지날수록 더 정확해짐

  • 태그 정규화
  • 누락 필드 보충
  • 중복 감지 + 정리
  • MOC/Companion 갱신

요약하면, 한 번 만든 지식을 “살아있는 지식”으로 유지.

가치

수동 점검 루틴이 줄어들고, 팀이 쓰기 쉬운 베이스가 된다.

사례: 신한은행 PoC

실전 검증: 포맷을 재사용할 수 있다

과거 자산

신한은행 PoC 발표 형식/구조를 그대로 사용

변환 방식

도메인만 교체, 전개 리듬은 그대로

이건 단순 재활용이 아니라 “반복 가능한 발표/분석 템플릿” 축적입니다.

개인 → 사내

사내 KB는 이 단계부터

1단계 (개인): 운영 리듬 정착

· 입력-자동정리-점검만 꾸준히

2단계 (팀/사내): 신뢰규칙 + 공개/비공개 구획

· 노이즈를 필터링 가능한 운영 규칙

핵심

“사내 KB를 먼저 만드는 것”보다 “사내가 받아들일 형태”를 먼저 만드는 게 우선이다.

한 번만 외우면 되는 결론

DotBrain이 강한 이유

  • 문제 해결: 지식은 있어도 AI가 못 쓰는 상태
  • 방법: 기능이 아니라 구조화한 운영 루프
  • 결과: 시간이 지날수록 재사용성과 정확도가 올라감

오늘의 문장: “새로운 AI를 기다리지 말고, 새롭게 사용할 수 있는 지식을 만들자.”
