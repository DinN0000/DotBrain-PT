“지식은 쌓였는데, AI는 잘 못 쓰는 시대”

15분 발표용 스크립트: 개인 지식운영을 통한 AI 활용 역량 강화

발표자: 화 / 2026-03

한 줄 핵심 : DotBrain은 복잡한 지식관리 시스템이 아니라, 지속 가능한 지식운영 루틴을 만드는 도구입니다.
문제 제기

우리는 이미 “리서치 자동화”는 하고 있다

  • 리서치 요청 → 여러 에이전트가 리포트 생성
  • 문서는 생기지만, 재사용 가능한 형태로 잘 관리되지 않음
  • AI는 필요할 때 그걸 열람해야 하지만, 실전에서는 매번 구조를 다시 뒤짐
즉, 생산은 늘었는데, 지속적 인사이트 생산성은 선형으로 늘지 않는다.
발표 멘트: “에이전트를 많이 돌려도, 결국 지식 조직화에서 시간을 다시 소비합니다.”
진입 장벽 분석

가장 큰 병목은 AI의 성능이 아니라, 운영 난이도

  • 요청은 쉬워도, 문서를 수집/정리/태깅/링킹은 시간이 걸림
  • 개인/팀 모두 “한 번 쓰고 잊는” 문서가 생김
  • 사내 도입 시 더 심각: 신뢰성 + 노이즈 관리가 병행돼야 함

실무에서 생기는 질문: “이 자료를 과연 다음에도 믿고 다시 쓸 수 있을까?”

제안

DotBrain은 사내 KB 이전의 개인 KB를 쉽게 만든다

기존 접근

  • 직접 정리 규칙을 새로 설계
  • 수작업 태그/링크/정리
  • 누군가가 관리하지 않으면 곧 무너짐

DotBrain 접근

  • 인박스에 넣으면 자동 분류/메타데이터
  • 사람은 마크다운 쓰기만 유지
  • AI가 읽을 수 있는 구조는 AI가 자동 보강

핵심 한마디: DotBrain은 처음부터 완성형 시스템을 요구하지 않는다. 작게 시작해서 확장한다.

핵심 주장 1

Auto Sort: 첫 진입 장벽을 없앤다

  • 새 파일/문서는 _Inbox/에 넣기만 하면 됨
  • AI가 PARA/메타데이터로 초기 정리
  • 사용자는 탐색과 분석에 집중 가능

기존: “분류할지 말지” 판단 비용이 큼 / DotBrain: “넣고 끄기”

핵심 주장 2

구조 자동화: AI가 즉시 컨텍스트를 얻는다

  • Frontmatter, MOC, Semantic links를 자동 보강
  • 관련 노트 후보를 점수 기반으로 연결
  • 요청 발생 시, 인덱스로 먼저 후보 필터링 → 정밀 참조

“필요할 때 필요 지식”을 찾는 시간이 크게 줄어들며, 동일 주제 재질문 시 일관성 향상

핵심 주장 3

Self-Healing: 시간이 지날수록 쓸모가 커진다

  • 태그 정규화, 누락 필드 보충, 중복 탐지
  • MOC·Companion 갱신 자동화
  • 매일 새 자료가 쌓여도 “깨끗한 상태” 유지

점점 더 “이전 문서를 다시 쓰지 않아도” 사용할 수 있는 상태가 된다.

구조화된 사례

신한은행 PoC 경험에서 다른 도메인으로의 재활용

신한은행 PoC 발표자료에 쓴 형식을, 다른 주제 발표 자료로도 같은 포맷·흐름으로 재생성했습니다.
  • 형식/톤을 유지하고 내용만 교체
  • 새 도메인 진입 시 제작 리드타임이 감소
  • 개인 지식자산이 팀 지식운영으로 전이되는 경로 형성

발표에서 이 부분은 “실전 검증 포인트”로 신뢰도 상승

운영 프레임

개인 지식베이스 → 사내 지식베이스로 연결

개인 단계

  • 개발 지식 없어도 사용
  • 인박스 처리, 자동 정리, 링크 보완
  • 출처·신뢰도 관리 습관 형성

사내 단계

  • 권한/공유 정책 적용
  • 검증된 템플릿 기반 출처 관리
  • 개인 기여 + 팀 신뢰성의 균형

핵심: 사내 KB를 바로 만들지 말고, 개인 운영 루틴으로 품질을 검증한 다음 확장

한줄 요약

왜 설득력 있는가?

  • 문제 정의가 분명하다: 지식은 있는데 재사용이 안 된다
  • 해결이 단순하다: 복잡한 도구가 아니라 구조 규칙
  • 효과가 축적된다: 시간이 지날수록 정확하고 빠르게

“새로 배우는 AI”보다 “기억할 수 있는 AI 워크플로”가 더 중요합니다.

데모/실행 흐름

실행 예시

  • 1) 새 자료 수집 → _Inbox
  • 2) AI 자동 분류/요약/메타데이터 생성
  • 3) 관련 노트 선별 후 새 요청 처리
  • 4) 하루/주간 Self-Healing로 품질 점검

“하루 한 번 돌아가는 루틴”으로 운영되면, 리포트 품질이 지속 개선됩니다.

마무리

다음 단계

  • DotBrain 기본 포맷 정착 (1주)
  • 개인 도메인 2~3개에 인덱스 운영 시작 (2주)
  • 성과 지표: 재검색 시간, 리서치 재생산 시간, 중복 리포트 비율 (4주)

지금부터 “무조건 더 큰 시스템”이 아니라, “지속 가능한 작은 시스템”으로 시작하면 됩니다.

이제 여러분이 만드는 AI 발표·리서치도, 구조를 갖추면 점점 더 빨라집니다.
